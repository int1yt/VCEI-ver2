# CAN-CNN(64x9) 方法描述（可直接用于论文）

## 1. 研究动机与任务定义

针对车载 CAN 总线入侵检测中“时序扰动与载荷篡改并存”的特征，本文采用二维张量建模方式，将连续报文片段映射为固定大小的 \(64\times9\) 输入，利用轻量卷积神经网络实现多类别攻击识别。该设计的核心思想是：在同一特征平面内联合编码报文载荷字节与相邻到达时间间隔（\(\Delta t\)），使模型同时具备对内容模式与时序扰动的感知能力。

与仅使用 payload 字节的静态特征方案相比，本方法显式引入了报文间时间关系；与仅依赖时钟偏移或到达周期统计的时序方法相比，本方法保留了字节级语义信息。因此，CAN-CNN(64x9) 的定位不是替代规则检测，而是提供一种“可学习的内容-时序联合判别器”，用于在复杂工况下提高攻击类型细分能力。

本文任务定义为四分类：
\[
\mathcal{Y}=\{\text{Normal},\ \text{DoS},\ \text{Fuzzy},\ \text{Spoofing}\}.
\]
其中 `gear` 与 `RPM` 相关样本统一映射为 `Spoofing` 类别，以降低稀疏标签导致的学习不稳定性。

该标签合并策略的工程意义在于：首先，避免小样本类别在训练中产生梯度不稳定和决策边界抖动；其次，使模型优先学习“正常/拒绝服务/模糊注入/伪造注入”这一更贴近在线部署需求的粗粒度攻击语义。

## 2. 输入构造与预处理流程

### 2.1 滑动窗口建模

设按时间排序后的 CAN 报文序列为 \(\{m_i\}_{i=1}^{N}\)，每条报文含时间戳 \(t_i\) 与最多 8 字节载荷。采用窗口长度 \(W=64\)、步长 \(S=32\) 的滑动策略生成样本：
\[
\mathbf{X}^{(k)}\in \mathbb{R}^{64\times9},\quad
k=1,2,\dots,\left\lfloor\frac{N-W}{S}\right\rfloor+1.
\]

其中 \(W=64\) 用于覆盖足够的局部时序上下文，\(S=32\) 用于在样本数量和相邻窗口冗余之间取得平衡。重叠滑窗可提升训练样本密度，并增强模型对攻击过渡阶段的捕获能力。

### 2.2 特征组织

对窗口内第 \(r\) 条报文（\(r\in[1,64]\)）：

- 第 1–8 列：8 字节载荷归一化值，\(x_{r,c}=b_{r,c}/255\)；
- 第 9 列：相邻报文时间间隔（毫秒）：
  \[
  \Delta t_r=
  \begin{cases}
  0, & r=1\\
  (t_r-t_{r-1})\times 1000, & r>1
  \end{cases}
  \]

为避免数据泄露，\(\Delta t\) 的归一化上界仅由训练集估计：
\[
\Delta t_{\max} = \max(\Delta t\_{\text{train}})+\epsilon,\qquad
\tilde{\Delta t}=\mathrm{clip}\!\left(\frac{\Delta t}{\Delta t_{\max}},0,1\right),
\]
并将 \(\Delta t_{\max}\) 持久化为 `preprocess_meta.json`，用于在线推理阶段保持训练-部署一致性。

这里“仅使用训练集估计归一化参数”的原则尤为关键：若在全数据范围上归一化，会将验证/测试分布信息泄露给训练过程，导致指标偏乐观；若部署端未复用相同 \(\Delta t_{\max}\)，则第 9 列特征将发生分布漂移，显著影响模型置信度校准与泛化性能。

### 2.3 数据划分

采用分层划分策略：70\% 训练、15\% 验证、15\% 测试，保证各类别在三个子集中的比例一致，降低类别不平衡对评估方差的影响。

## 3. 网络结构设计

模型为轻量 CNN，输入尺寸为单通道 \(1\times64\times9\)。网络由 3 个卷积块与一个全连接分类头组成：

1. Conv(1→32, \(3\times3\), padding=1) + BN + ReLU + MaxPool(2)  
2. Conv(32→64, \(3\times3\), padding=1) + BN + ReLU + MaxPool(2)  
3. Conv(64→128, \(3\times3\), padding=1) + BN + ReLU + MaxPool(2)  
4. AdaptiveAvgPool(\(2\times2\)) + Dropout(0.5) + Linear(512→4)

输出层直接给出 logits，与交叉熵损失配合训练：
\[
\mathcal{L}_{CE}=-\sum_{c=1}^{4} y_c\log \hat{p}_c.
\]

采用自适应池化的目的是稳定空间维度，减少输入尺度变化或边界效应对分类头参数规模的影响。

从表示学习角度看，前两层卷积主要抽取局部字节模式与短程时间纹理，第三层卷积进一步形成更抽象的攻击判别表示。`AdaptiveAvgPool(2x2)` 将空间特征压缩到固定维度，有利于在不显著增加参数量的前提下维持判别能力；`Dropout(0.5)` 则用于抑制过拟合，提高跨数据分片的稳定性。

## 4. 训练与模型选择策略

优化器采用 Adam（初始学习率 \(1\times10^{-3}\)），并使用 `ReduceLROnPlateau` 按验证损失自适应降学习率（factor=0.5, patience=3）。训练过程中记录 `train_loss`、`val_loss` 与 `val_acc`，并以最小验证损失选择最优权重 `best_model.pth`。

相较于固定学习率策略，`ReduceLROnPlateau` 能在验证损失进入平台期后自动降低步长，兼顾收敛速度与后期细化能力。模型选择使用 `val_loss` 而非 `val_acc` 的原因是：损失函数对概率分布变化更敏感，能够更早反映过拟合趋势，便于选取泛化性能更稳健的 checkpoint。

测试阶段使用分批推理以控制显存/内存峰值，输出指标包括：

- Accuracy
- Macro Precision / Recall / F1
- Confusion Matrix
- Classification Report

上述结果统一写入 `eval_metrics.json` 与 `classification_report.txt`，便于论文复现实验。

在实际报告中，建议同时给出宏平均（macro）与加权平均（weighted）指标：前者更能反映少数类表现，后者更贴近整体运行效果。二者联合可避免单一准确率掩盖类别不平衡问题。

## 5. 在线部署与系统集成

在系统部署中，`ml_bridge` 优先加载 `can_cnn_64x9` 权重（若存在），并从 `preprocess_meta.json` 读取 `dt_max_ms` 对实时 \(\Delta t\) 列归一化。实时数据由守护进程提供至少 64 条 CAN 历史窗口，转换为 \(64\times9\) 后输入模型，输出：

\[
(\text{class\_id},\ \text{class\_name},\ \text{confidence},\ \text{class\_probs}).
\]

该输出可与规则引擎结果联合用于最终告警融合，提升攻击类型细分能力。

系统层面建议将 CAN-CNN 输出作为“高语义增量信息”接入告警管道：当规则检测已触发异常时，模型输出用于补充分型；当规则证据较弱时，模型概率可作为辅助排序依据，而非单独决定最终告警级别，从而降低误报风险。

## 6. 复杂度与工程可行性

设单样本输入大小为 \(64\times9\)，卷积核尺寸固定，网络深度常数级，则单次推理复杂度可写为
\[
O\!\left(\sum_{l=1}^{3} H_lW_lC_lK^2\right),
\]
在当前轻量配置下满足车端近实时推理需求。与更深模型相比，该结构在参数量、推理时延与鲁棒性之间取得工程平衡，适合部署于资源受限的车载计算环境。

从工程实现视角，CAN-CNN(64x9) 的主要开销集中在离线训练阶段；在线阶段仅包含“窗口构造 + 归一化 + 单次前向传播”，其资源需求显著低于大规模时序 Transformer。该特性使其能够在边缘算力受限场景下稳定运行，并与现有 C++ 检测链路低耦合集成。

## 7. 局限性与可扩展方向

当前模型以固定窗口捕获局部时序结构，对超长周期依赖建模能力有限；同时，类别映射将部分语义（如 `gear` 与 `RPM` 细分）合并到 `Spoofing`，有助于稳定训练但牺牲了细粒度区分。后续可探索：

- 多尺度窗口并行编码（短窗 + 长窗）；
- 轻量时序注意力模块增强长依赖建模；
- 分层分类头实现“粗类检测 + 细类判别”。

此外，当前输入未显式编码 CAN ID 语义与跨 ID 交互关系，模型主要学习窗口内部统计结构。后续可通过“ID 嵌入 + 通道注意力”或“图结构建模”进一步增强跨报文来源关系建模能力。
