# 以太网环形存储机制（可直接用于论文）

## 1. 设计目标与系统定位

在车载跨域入侵检测场景中，CAN 总线侧告警通常仅反映时序异常本身，难以单独说明攻击是否伴随以太网侧先导行为。为提高告警可解释性，本文在中央处理器中引入以太网环形存储（Ethernet Ring Buffer），用于维护最近一段时间的以太网报文上下文，并在 CAN 异常触发时执行时间窗关联检索，从而实现“时序异常 + 网络上下文”的联合证据构建。

该机制不依赖离线回放，采用在线增量更新，满足车内实时处理对内存上限、查询时延和实现复杂度的约束。

## 2. 数据结构定义

设环形缓冲容量为 \(C\)，以太网报文序列项定义为
\[
e_i = (t_i, s_i, d_i, p_i, l_i, f_i),
\]
其中 \(t_i\) 为毫秒时间戳，\(s_i/d_i\) 分别表示源/目的地址，\(p_i\) 为协议类型，\(l_i\) 为负载长度，\(f_i\) 为可选标志位。系统维护如下状态：

- `buffer[0...C-1]`：固定长度存储区；
- `head`：下一次写入位置；
- `tail`：当前最旧元素位置；
- `count`：当前有效元素个数，且 \(0 \le count \le C\)。

该结构满足固定内存占用 \(O(C)\)，避免动态扩容带来的不可控时延。

## 3. 写入与覆盖策略

对于新到达报文 \(e_{new}\)，执行：
\[
buffer[head] \leftarrow e_{new}, \quad
head \leftarrow (head + 1)\bmod C.
\]

若 `count < C`，则 `count` 自增；否则说明缓冲区已满，需要同时推进
\[
tail \leftarrow (tail + 1)\bmod C,
\]
即覆盖最旧报文，实现“新数据优先保留”的滑动历史窗口。

该策略的核心优势在于单次写入时间复杂度为 \(O(1)\)，且不会因高流量突发而触发额外内存分配。

## 4. 时间窗检索机制

当 CAN ClockIDS 在时刻 \(t_a\) 触发异常时，中央处理器构造检索时间窗
\[
\mathcal{W}(t_a) = [t_a-\Delta_{pre},\; t_a+\Delta_{post}],
\]
并在环形缓冲有效区间内顺序扫描，返回满足
\[
t_i \in \mathcal{W}(t_a)
\]
的全部以太网报文。实现层面采用“从 `tail` 到 `tail + count - 1`（模 \(C\)）”的逻辑顺序遍历，保证结果在时间上保持由旧到新的可解释顺序。

需要强调的是，时间窗判断采用闭区间（inclusive）：
\[
t_i \ge t_{start} \land t_i \le t_{end}.
\]
该定义可避免边界时间戳在离散毫秒量化下被误丢弃。

## 5. 与 CAN 异常检测的关联逻辑

系统运行时，CAN 与以太网流分别增量进入中央处理链路：以太网报文持续写入环形缓冲，CAN 报文则进入 ClockIDS 时序判定。一旦 CAN 侧给出异常结论，系统立即从环形缓冲提取对应时间窗的上下文报文集 \(\mathcal{E}_a\)，并将其附加到告警载荷中。

该机制提供两类关键收益：

1. **告警可解释性提升**：可直接观察异常前后是否存在扫描、连接突增、协议切换等网络征兆；
2. **跨域关联能力增强**：将“控制域时序异常”与“以太网行为线索”对齐，为后续融合分类提供证据基础。

## 6. 复杂度与实时性分析

- **写入复杂度**：\(O(1)\)；
- **空间复杂度**：\(O(C)\)；
- **单次查询复杂度**：最坏 \(O(C)\)，在窗口较窄且有效数据稀疏时平均代价低于最坏情形。

在车载边缘计算场景下，该复杂度组合具有较好的工程可部署性：写路径恒定时延，读路径仅在告警触发时执行，不对主路径造成持续性抖动。

## 7. 参数选型建议（论文可写）

为兼顾上下文完整性与检索开销，容量 \(C\) 与时间窗参数 \((\Delta_{pre}, \Delta_{post})\) 建议按“链路峰值报文率 \(\times\) 覆盖时长”联合标定：
\[
C \ge r_{\max} \cdot T_{ctx},
\]
其中 \(r_{\max}\) 为峰值报文率，\(T_{ctx}\) 为期望保留时长。若系统更关注攻击前导行为，可取 \(\Delta_{pre}>\Delta_{post}\)；若关注攻击后果扩散，则可反向配置。

## 8. 方法局限与改进方向

当前环形存储主要完成“时间邻域关联”，尚未在缓冲层引入协议语义压缩与优先级索引；在极端高负载下，最坏查询仍受 \(O(C)\) 线性扫描约束。后续可考虑：

- 增加按时间分桶或多级索引以降低查询代价；
- 引入协议感知采样，优先保留与安全相关字段；
- 在保持固定内存预算前提下实现分层环形缓冲（短窗高分辨率 + 长窗低分辨率）。
